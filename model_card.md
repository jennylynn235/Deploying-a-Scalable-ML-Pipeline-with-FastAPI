# Model Card

For additional information, see the Model Card paper: https://arxiv.org/pdf/1810.03993.pdf

## Model Details

This project implements a binary classification model using a Random Forest Classifier to predict whether an individual's annual income exceeds $50,000, based on demographic and employment-related features from the U.S. Census Income dataset. The machine learning pipeline includes data preprocessing, model training, model serialization, performance evaluation, and inference through a FastAPI application. Categorical features are encoded using One-Hot Encoding, and the target variable is transformed using a Label Binarizer. A Random Forest Classifier was selected because it performs well on structured tabular data, requires minimal feature scaling, and can model complex, non-linear relationships while remaining relatively robust to overfitting through the use of multiple decision trees.

## Intended Use

The model is intended for educational purposes as part of the Udacity Machine Learning DevOps Engineer Nanodegree. It demonstrates the complete lifecycle of a machine learning project, including preprocessing, model training, evaluation, testing, deployment, and model monitoring through slice-based performance analysis. It is not intended for use in real-world hiring, lending, employment, or other high-impact decision-making systems.

## Training Data

The model was trained using the publicly available U.S. Census Income dataset from the UCI Machine Learning Repository. The dataset contains demographic and employment-related information, including age, education, occupation, marital status, work class, race, sex, hours worked per week, and native country. The target variable indicates whether an individual's annual income is greater than $50,000.

Prior to training, categorical features were encoded with a One-Hot Encoder, and the target labels were transformed with a Label Binarizer.

## Evaluation Data

The dataset was randomly split into training and testing subsets using an 80/20 train-test split with a fixed random state to ensure reproducibility. Model performance was evaluated on the held-out test dataset. Additional evaluation was performed by measuring performance across individual categorical feature values to better understand how the model performs on different subsets of the data.

## Metrics

Model performance was evaluated using Precision, Recall, and F1 Score to assess the classifier's predictive performance on the held-out test dataset."

**Overall model performance**
- Precision: 0.7419
- Recall: 0.6384
- F1 Score: 0.6863

In addition to overall performance, slice-based metrics were generated for every categorical feature value to identify differences in model performance across demographic and employment-related groups.

## Ethical Considerations

Although this dataset is widely used for educational purposes, it contains sensitive demographic attributes such as race, sex, and native country. Models trained on this data may learn historical patterns and biases present in the original dataset. Predictions generated by this model should therefore not be used to make decisions that impact individuals without additional fairness analysis, bias testing, and ongoing monitoring. 

## Caveats and Recommendations

This model was developed as an educational demonstration rather than a production-ready system. Several improvements could be made in future iterations, including hyperparameter tuning, cross-validation, additional feature engineering, fairness assessments, and more comprehensive model monitoring. Future versions could also include automated retraining, experiment tracking, and continuous deployment to improve maintainability and performance. Any production deployment should also include ongoing monitoring to detect data drift and changes in model performance over time.
